Section A

Theory — Probability & Statistics

Random Variables
Definition A random variable is a variable whose value is determined by the outcome of a random experiment. You don't know what value it will take before the experiment, but every possible outcome has a probability attached to it.

Think of it like rolling a die. The number you'll get is uncertain before you roll — it could be 1, 2, 3, 4, 5, or 6. Each of those is a possible value of the random variable X. We write X = 3 to mean "the outcome turned out to be 3."

In Econometrics, almost everything is a random variable — income, spending, test scores, GDP growth. We use them to model real-world uncertainty.

Two Types of Random Variables

Discrete Random Variable Takes a countable set of values. There are gaps between the values (you can list them). Example: number of children in a family (0, 1, 2, 3…). You can't have 1.5 children.
Continuous Random Variable Takes any value within a range — no gaps. Example: income (could be R14 285.37). You cannot list all possible values.

Probability Distributions

A probability distribution tells you how likely each value (or range of values) is.

  • For discrete variables: we use a Probability Mass Function (PMF). It gives the exact probability of each value: P(X = x).
  • For continuous variables: we use a Probability Density Function (PDF). Since the variable is continuous, we can't give the probability of an exact value — instead we find the probability of a range by calculating the area under the curve.
Memory Trick Discrete = Dots (you can count them). Continuous = Curve (smooth, no gaps). PMF = Precise, PDF = Area.

Key Rules for Probability Distributions

  • All probabilities must be ≥ 0 (you can't have a negative chance).
  • All probabilities must sum to 1 (something must happen).
  • For continuous: the total area under the PDF curve = 1.
Common Mistake Students confuse the PDF value with a probability. The height of the PDF curve at a point is NOT the probability of that exact point. The probability of a single point in a continuous distribution is always zero. You need an interval (area) to get a probability.

Joint, Marginal, and Conditional Distributions

Joint Probability Distribution Describes the probability of two variables occurring together simultaneously. Written as P(X = x, Y = y) or f(x, y). It answers: "What's the probability that X takes value x AND Y takes value y at the same time?"
Marginal Probability Distribution Gives the probability distribution of ONE variable, ignoring the other. You get it by summing (discrete) or integrating (continuous) the joint distribution over all values of the other variable.

For discrete: P(X = x) = Σ P(X = x, Y = y) for all y
Conditional Probability The probability of event A occurring, given that event B has already occurred.

P(A|B) = P(A and B) / P(B)

In words: "The probability of A, given B" = the probability they both occur, divided by the probability B occurs.

Example: What is the probability a student passes (A), given they studied (B)?

P(Pass | Studied) = P(Pass AND Studied) / P(Studied)

Expected Value, Variance & Standard Deviation
Expected Value E(X) The long-run average value of a random variable. If you repeated the experiment thousands of times, the average outcome would approach E(X). It is NOT necessarily a value the variable will actually take — it's the weighted average of all possible values.

For a discrete variable: E(X) = Σ [x · P(X = x)]
You multiply each value by its probability, then add everything up.

Properties of Expected Value (MEMORISE THESE)

  • E(c) = c — The expected value of a constant is just that constant. If something never changes, you know exactly what it is.
  • E(cX) = c·E(X) — A constant multiplier comes out of the expectation.
  • E(X + Y) = E(X) + E(Y) — The expected value of a sum equals the sum of expected values. Always true, regardless of independence.
  • E(XY) = E(X)·E(Y) — Only true if X and Y are independent. Very important caveat!
  • E(aX + b) = aE(X) + b — Combines the scaling and shift rules.
Variance Var(X) Measures how spread out the values of X are around the mean. A high variance means values can be very far from the average. A low variance means values cluster closely around the average.

Var(X) = E[(X − μ)²] = E(X²) − [E(X)]²

The second formula (E(X²) minus the square of E(X)) is the one you'll use most in calculations.

Properties of Variance (MEMORISE THESE)

  • Var(c) = 0 — A constant has no variability. It never changes.
  • Var(cX) = c²·Var(X) — Note it's c squared. If you double X, variance becomes 4 times larger.
  • Var(X + Y) = Var(X) + Var(Y) — Only true if X and Y are independent.
  • Var(X + c) = Var(X) — Adding a constant shifts the distribution but doesn't change its spread.
  • Var(aX + b) = a²·Var(X) — The constant b disappears because shifting doesn't affect spread.
Standard Deviation Simply the square root of variance. We use it because it is in the same units as the original variable (variance is in squared units, which is hard to interpret).

σ = √Var(X)
Memory Trick — Var vs SD Variance = Spread². Standard Deviation = Spread. SD is more interpretable because it is in the same units as your data.
Common Mistake Students forget that Var(X + Y) = Var(X) + Var(Y) only holds for INDEPENDENT variables. If X and Y are correlated, you need to add the covariance term: Var(X + Y) = Var(X) + Var(Y) + 2·Cov(X,Y).
The Four Key Distributions

1. The Normal Distribution N(μ, σ²)

Definition The bell-shaped, symmetric distribution that describes many natural phenomena. Fully described by its mean (μ) and variance (σ²). The standard normal has μ = 0 and σ² = 1, written Z ~ N(0, 1).

Key properties:

  • Symmetric around the mean. Mean = Median = Mode.
  • 68% of values fall within 1 standard deviation of the mean.
  • 95% of values fall within 1.96 standard deviations (critical value for 5% significance).
  • 99% of values fall within 2.576 standard deviations.
  • The tails extend to infinity but approach zero.

Standardising: To convert any normal variable X ~ N(μ, σ²) to a standard normal Z, use: Z = (X − μ) / σ

2. Student's t Distribution

Definition A bell-shaped distribution like the normal, but with heavier (fatter) tails. It is used in hypothesis testing when the sample size is small and/or the population variance is unknown (which is almost always in practice). It is defined by its degrees of freedom (df).

Key properties:

  • As degrees of freedom increase, the t-distribution approaches the normal distribution.
  • With df > 120 (large samples), t and z critical values are essentially the same.
  • Degrees of freedom = n − k, where n is sample size and k is the number of estimated parameters.
  • For simple linear regression: df = n − 2 (we estimate β₀ and β₁).
Memory Trick t-distribution = Normal distribution with fat tails. Fat tails = more uncertainty. More uncertainty = larger critical values = harder to reject H₀. As sample size grows, fat tails disappear → becomes normal.

3. Chi-Square Distribution (χ²)

Definition A right-skewed distribution that arises when you square independent standard normal variables. If Z ~ N(0,1), then Z² ~ χ²(1). The chi-square with k degrees of freedom is the sum of k squared standard normal variables.

Key uses in Econometrics:

  • It forms the basis of the F-distribution.
  • Used in testing overall model fit.
  • Always positive (squares can't be negative).
  • Skewed right, not symmetric.

4. F Distribution

Definition The F-distribution is the ratio of two independent chi-square variables divided by their respective degrees of freedom. It is always positive and right-skewed.

F = (χ²₁/df₁) / (χ²₂/df₂)

Key uses:

  • F-test for overall significance: Is the entire regression model statistically significant? (Are ALL slope coefficients zero?)
  • F-test for joint significance: Are specific groups of coefficients jointly zero?
  • Defined by two sets of degrees of freedom: numerator df and denominator df.
Memory Trick — Distributions and Their Uses
Normal → population testing when σ² known (rare in practice)
t → coefficient testing (β₀, β₁) — you'll use this most
χ² → squared things, building block for F
F → overall model / joint significance testing
Skewness & Kurtosis
Skewness Measures the asymmetry of a distribution. A perfectly symmetric distribution (like the normal) has skewness = 0.

• Positive skew (right-skewed): The tail stretches to the RIGHT. Mean > Median. Income distributions are typically right-skewed.
• Negative skew (left-skewed): The tail stretches to the LEFT. Mean < Median.
Kurtosis Measures the thickness of the tails (how extreme the outliers are). The normal distribution has kurtosis = 3. We often use excess kurtosis = kurtosis − 3.

• Leptokurtic (excess kurtosis > 0): Fatter tails than normal → more extreme values. (Think: t-distribution)
• Platykurtic (excess kurtosis < 0): Thinner tails than normal → fewer extreme values.
• Mesokurtic (excess kurtosis = 0): Normal distribution.
Memory Trick Lepto = Lean (tall, thin peak, fat tails). Platy = Flat (flat peak, thin tails). Like a platypus — wide and flat.
Type I and Type II Errors

When we test a hypothesis, we make a decision based on sample data. Since data is random, we can make two kinds of errors:

DecisionH₀ is Actually TRUEH₀ is Actually FALSE
Reject H₀Type I Error (α)Correct Decision ✓
Fail to Reject H₀Correct Decision ✓Type II Error (β)
Type I Error Rejecting H₀ when it is actually TRUE. This is a false positive. The probability of making a Type I error is α (the significance level). If α = 0.05, there is a 5% chance of incorrectly rejecting a true H₀.
Type II Error Failing to reject H₀ when it is actually FALSE. This is a false negative. The probability is β. Power of the test = 1 − β (the probability of correctly rejecting a false H₀).
Memory Trick Type I = Crying wolf (you said the wolf came, but it didn't). Type II = Missing the wolf (it came, but you didn't notice).
Common Mistake Students say "we accept H₀" when they should say "we fail to reject H₀." Failing to find evidence against H₀ is not the same as proving H₀ is true. We never truly "accept" — we only "fail to reject." This distinction matters in exams.

Small Sample vs Large Sample Properties

Small Sample Properties Properties of an estimator that hold regardless of sample size. The most important: unbiasedness — E(β̂) = β. If an estimator is unbiased, it doesn't systematically over- or under-estimate the true parameter.
Large Sample (Asymptotic) Properties Properties that only kick in as sample size n → ∞.
• Consistency: As n grows, the estimator converges to the true value. β̂ → β.
• Asymptotic normality: Even if errors aren't normal, the OLS estimators become normally distributed as n → ∞ (Central Limit Theorem).

Theory — CLRM & OLS

The Classical Linear Regression Model (CLRM)
What is the CLRM? The Classical Linear Regression Model is the foundational framework of econometrics. It describes the relationship between a dependent variable (Y) and one or more independent variables (X), using a linear equation. The word "classical" refers to a specific set of assumptions — if these hold, OLS gives us the best possible estimates.

Population Regression Function (PRF)

PRF The true, unknown relationship in the entire population. We never observe this directly — we can only estimate it from sample data.

Y = β₀ + β₁X + ε

• Y = dependent variable (what we're trying to explain)
• X = independent / explanatory variable (what we think drives Y)
• β₀ = true population intercept (unknown)
• β₁ = true population slope (unknown)
• ε (epsilon) = error term / disturbance term

The Error Term ε — Why It Exists

No model is perfect. The error term captures everything that affects Y but is NOT included in the model:

  • Variables we forgot to include (omitted variables)
  • Measurement errors in Y
  • Truly random human behaviour that can't be predicted
  • Model approximation error

Sample Regression Function (SRF)

SRF Our estimate of the PRF, computed from the data we have. Because we use sample data, we get estimates (denoted with hats) rather than true population values.

Ŷ = β̂₀ + β̂₁X

• Ŷ (Y-hat) = predicted/fitted value of Y
• β̂₀ = estimated intercept
• β̂₁ = estimated slope
• The SRF is what you calculate and what appears in your regression output.

Residuals vs Errors

Residual (ê or û) The difference between the ACTUAL value of Y and the PREDICTED value from the SRF.

ê = Y − Ŷ = Y − (β̂₀ + β̂₁X)

Residuals are observable (we can calculate them from data). The true error ε is unobservable.
Common Mistake Students use "error" and "residual" interchangeably. They are different. ε is the unobservable population error. ê is the observable sample residual. OLS minimises the sum of squared residuals, not the sum of squared errors.
CLRM Assumptions (Memorise All of These)

These are the conditions under which OLS is the Best Linear Unbiased Estimator (BLUE). If any assumption is violated, OLS may not be optimal and you need to address the problem.

Assumption 1: Linear in Parameters

The relationship between Y and X must be linear in the parameters (β₀ and β₁). The model Y = β₀ + β₁X satisfies this. Note: X itself doesn't need to be linear — log(X) or X² is allowed as long as the betas appear linearly.

Assumption 2: X Values are Fixed (Non-Stochastic X)

In repeated sampling, the values of X stay the same. X is not random — it is fixed or determined outside the model. This means X and ε are independent.

Assumption 3: Zero Mean of Error Term

E(ε | X) = 0. The average value of the error, given any value of X, is zero. This means our model is not systematically wrong. On average, the errors cancel out.

Common Mistake This does NOT mean every residual is zero. It means they average to zero.

Assumption 4: Homoscedasticity (Constant Variance)

Var(ε | X) = σ² = constant for ALL values of X. The spread of errors is the same across all observations. When this is violated, we have heteroscedasticity.

Heteroscedasticity When the variance of the error term is NOT constant — it changes with X. Common in data where larger values of X naturally have more spread (e.g., income and spending). Consequence: OLS is still unbiased but no longer efficient (not BLUE).

Assumption 5: No Autocorrelation

Cov(εᵢ, εⱼ) = 0 for i ≠ j. The errors for different observations are uncorrelated. Common violation in time series data where this period's error affects next period's error.

Assumption 6: No Perfect Multicollinearity

No exact linear relationship between the independent variables. In simple regression (one X), this is automatically satisfied. In multiple regression, if two X variables are perfectly correlated, we cannot separately estimate their effects.

Assumption 7: Normality of Errors

ε ~ N(0, σ²). The error term follows a normal distribution. This is needed for t-tests and F-tests to be valid in small samples. In large samples, the Central Limit Theorem makes this less critical.

Memory Trick — Remember the Assumptions "LZCHNN" — Linear, Zero mean, Constant variance, Homoscedasticity, No autocorrelation, Normality. Or think: "Every Good Regression Assumes Normally Distributed Noise" — just remember what each word stands for.
Ordinary Least Squares (OLS)
What is OLS? Ordinary Least Squares is the method we use to estimate the parameters β₀ and β₁. The idea is simple: draw a line through the data that is as close to all the data points as possible. "As close as possible" means minimising the sum of squared residuals.

OLS finds the values of β̂₀ and β̂₁ that minimise:

OLS Objective — Minimise SSR = Σ ê² = Σ (Yᵢ − Ŷᵢ)² = Σ (Yᵢ − β̂₀ − β̂₁Xᵢ)²

Why square the residuals? Because without squaring, positive and negative errors would cancel out, giving a misleadingly small sum. Squaring also penalises large errors more heavily than small ones.

OLS Properties — BLUE

BLUE — Best Linear Unbiased Estimator The Gauss-Markov Theorem states that, under the CLRM assumptions (excluding normality), OLS is BLUE. Let's break that down:

• Best: Has the smallest variance among all linear unbiased estimators. No other linear unbiased method gives more precise estimates.
• Linear: The estimator is a linear function of the dependent variable Y.
• Unbiased: E(β̂₁) = β₁. On average, across all possible samples, our estimate equals the true value.
• Estimator: It's a rule for computing estimates from data.
Memory Trick — BLUE Best = lowest variance. Unbiased = not systematically wrong. Linear = computed as a weighted sum of Y values. If assumptions hold → OLS gives you BLUE. If assumptions fail → OLS might still be unbiased but loses "Best."

Theory — Functional Forms & Interpretation

The Four Regression Model Types

Econometrics allows us to model relationships that are not straight lines by transforming variables using logarithms. There are four main functional forms, each with a different interpretation of the slope coefficient.

ModelEquationSlope InterpretationWhen to Use
Lin-Lin Y = β₀ + β₁X ΔY = β₁·ΔX
(unit change)		 Relationship is linear; both Y and X are in original units
Log-Lin ln(Y) = β₀ + β₁X %ΔY ≈ β₁·100·ΔX
(semi-elasticity)		 Y grows at a constant % rate as X changes by 1 unit
Lin-Log Y = β₀ + β₁·ln(X) ΔY ≈ (β₁/100)·%ΔX
(semi-elasticity)		 Y changes by fixed amount for each % change in X
Log-Log ln(Y) = β₀ + β₁·ln(X) %ΔY = β₁·%ΔX
(elasticity)		 Constant elasticity model — % relationship throughout
Memory Trick — Which Side Has the Log? Log-Lin: Log on LEFT (Y), Lin on RIGHT (X). → interpret as % change in Y for unit change in X. Lin-Log: Lin on LEFT (Y), Log on RIGHT (X). → interpret as unit change in Y for % change in X. Log-Log: Both sides log. → elasticity interpretation. β₁ IS the elasticity directly.

Interpreting the Intercept

The intercept β̂₀ is the predicted value of Y (or ln(Y)) when X = 0. However, X = 0 may not make practical sense in many models. When that's the case, the intercept simply ensures the regression line passes through the means (X̄, Ȳ) — it is a mathematical anchor, not always economically meaningful.

Dummy Variables

Dummy Variable A binary variable that takes only the value 0 or 1, used to represent qualitative categories (e.g., gender: Male = 1, Female = 0; or location: Urban = 1, Rural = 0).

Model: Y = β₀ + β₁X + β₂D + ε

• When D = 0: Ŷ = β̂₀ + β̂₁X (base group)
• When D = 1: Ŷ = (β̂₀ + β̂₂) + β̂₁X (dummy group)

β̂₂ represents the vertical shift in the regression line between the two groups. It is the average difference in Y between D=1 and D=0, holding X constant.
Common Mistake — Dummy Variable Trap If you have k categories, include only k−1 dummy variables. If you include all k, you have perfect multicollinearity (the dummies sum to 1, which equals the intercept column). Always leave one category as the base/reference group.

Theory — Hypothesis Testing

The Logic of Hypothesis Testing

Hypothesis testing is the process of using sample evidence to make decisions about population parameters. Here is the step-by-step logic, which you must follow every time:

Step-by-Step Procedure

  1. State H₀ (Null Hypothesis): The claim we are testing. Usually states "no effect" or "no significance." Example: H₀: β₁ = 0 (X has no effect on Y).
  2. State H₁ (Alternative Hypothesis): What we believe is true if H₀ is wrong. Could be two-tailed (≠) or one-tailed (> or <).
  3. Choose significance level α: Typically α = 0.05 (5%). This sets how much risk of Type I error we accept.
  4. Calculate the test statistic: For testing β₁, calculate the t-statistic.
  5. Find the critical value: Look up t-table using degrees of freedom (n − 2 for simple regression) and α.
  6. Decision rule: If |t-stat| > t-critical → Reject H₀. If |t-stat| ≤ t-critical → Fail to reject H₀.
  7. Conclusion: State what the result means in context.

Two-Tailed vs One-Tailed Tests

Two-Tailed Test H₀: β₁ = 0 vs H₁: β₁ ≠ 0
We reject if the coefficient is significantly different from zero in EITHER direction (positive or negative). Split α equally: α/2 in each tail.
Critical value at 5%: ±t(α/2, df) = ±t(0.025, df)
One-Tailed Test H₀: β₁ = 0 vs H₁: β₁ > 0 (right-tailed) or H₁: β₁ < 0 (left-tailed)
We only care about one direction. All of α goes in one tail.
Critical value at 5%: t(α, df) = t(0.05, df)
Memory Trick Two-tailed: "not equal" in H₁ → divide alpha in two. One-tailed: "greater than" or "less than" in H₁ → all alpha in one tail. Two-tailed critical values are always LARGER (harder to reject) than one-tailed.

The F-Test for Overall Model Significance

F-Test Tests whether the regression model as a whole is statistically significant. Asks: "Are ALL slope coefficients jointly equal to zero?" If yes, the model has no explanatory power at all.

H₀: β₁ = β₂ = ... = βₖ = 0 (no variable matters)
H₁: At least one βⱼ ≠ 0

Decision: If F-stat > F-critical → Reject H₀ → model is significant.
Common Mistake Significant F-test does NOT mean every individual variable is significant. It only means at least one is. You still need individual t-tests to identify which variables matter.

Confidence Intervals

Confidence Interval A range of plausible values for the true population parameter. A 95% CI means: if we repeated sampling many times, 95% of the constructed intervals would contain the true β.

CI = β̂ ± t(α/2, df) × SE(β̂)
Connection: CI and Hypothesis Test If H₀: β₁ = 0 and zero is NOT contained in the 95% CI → Reject H₀ at 5% significance. The CI and the two-tailed t-test always agree!

Complete Formula Sheet

OLS Estimators
Slope Estimator β̂₁ β̂₁ = Σ(Xᵢ − X̄)(Yᵢ − Ȳ) / Σ(Xᵢ − X̄)²

Equivalently: β̂₁ = [ΣXᵢYᵢ − n·X̄·Ȳ] / [ΣXᵢ² − n·X̄²]

Or using deviation form: β̂₁ = Σxᵢyᵢ / Σxᵢ² where xᵢ = Xᵢ − X̄ and yᵢ = Yᵢ − Ȳ
Intercept Estimator β̂₀ β̂₀ = Ȳ − β̂₁·X̄

IMPORTANT: Always calculate β̂₁ FIRST, then use it to find β̂₀
Sample Regression Function Ŷᵢ = β̂₀ + β̂₁Xᵢ
Variance and Standard Error of β̂₁
Variance of the Regression (σ̂²) σ̂² = SSR / (n − 2) = Σê² / (n − 2)

where SSR = Σ(Yᵢ − Ŷᵢ)² = Σê² = sum of squared residuals n − 2 = degrees of freedom (we estimated β̂₀ and β̂₁)
Variance of β̂₁ Var(β̂₁) = σ̂² / Σ(Xᵢ − X̄)² = σ̂² / Σxᵢ²
Standard Error of β̂₁ SE(β̂₁) = √Var(β̂₁) = √[σ̂² / Σxᵢ²]
Goodness of Fit
Sum of Squares Decomposition TSS = ESS + SSR

TSS = Total Sum of Squares = Σ(Yᵢ − Ȳ)² [total variation in Y] ESS = Explained Sum of Squares = Σ(Ŷᵢ − Ȳ)² [variation explained by model] SSR = Sum of Squared Residuals = Σ(Yᵢ − Ŷᵢ)² [unexplained variation]
Coefficient of Determination R² R² = ESS / TSS = 1 − (SSR / TSS)

Range: 0 ≤ R² ≤ 1 R² = 0 → model explains nothing R² = 1 → model explains everything perfectly
Correlation Coefficient r r = Σ(Xᵢ − X̄)(Yᵢ − Ȳ) / √[Σ(Xᵢ − X̄)² · Σ(Yᵢ − Ȳ)²]

Range: −1 ≤ r ≤ 1 Relationship with R²: r² = R² (in simple regression ONLY) Therefore: r = ±√R² Sign of r = sign of β̂₁
Common Mistake r² = R² only holds in SIMPLE regression (one X variable). In multiple regression, you cannot take the square root of R² to get r.
Hypothesis Testing
t-Statistic for β̂₁ t = β̂₁ / SE(β̂₁)

Under H₀: β₁ = 0, this follows a t-distribution with df = n − 2

For general H₀: β₁ = β₁* t = (β̂₁ − β₁*) / SE(β̂₁)
Confidence Interval for β̂₁ CI = β̂₁ ± t(α/2, n−2) × SE(β̂₁)

Lower bound: β̂₁ − t_crit × SE(β̂₁) Upper bound: β̂₁ + t_crit × SE(β̂₁)
F-Statistic for Overall Significance F = ESS/k / SSR/(n−k−1)

where: k = number of slope coefficients (= 1 for simple regression) n−k−1 = denominator degrees of freedom

For simple regression (k=1): F = ESS/1 / SSR/(n−2)

Also: F = [R²/k] / [(1−R²)/(n−k−1)]

In simple regression: F = t² (F-stat equals t-stat squared)
Elasticities
Point Elasticity (Lin-Lin Model) ε = β̂₁ × (X̄ / Ȳ)

OR at any specific point (X₀, Y₀): ε = β̂₁ × (X₀ / Y₀)
Log-Log Model Elasticity ε = β̂₁ (directly — no calculation needed)
Log-Lin Model: % Change in Y for unit change in X %ΔY ≈ β̂₁ × 100
Lin-Log Model: Unit change in Y for 1% change in X ΔY ≈ β̂₁ / 100
Probability Formulas
Expected Value (Discrete) E(X) = Σ xᵢ · P(X = xᵢ)
Variance Var(X) = E(X²) − [E(X)]² E(X²) = Σ xᵢ² · P(X = xᵢ)
Conditional Probability P(A|B) = P(A ∩ B) / P(B)
Standardisation (Z-score) Z = (X − μ) / σ ~ N(0, 1)

Step 4 — Calculate β̂₀ β̂₀ = Ȳ − β̂₁·X̄ = 8 − 1.05 × 6 = 8 − 6.3 = 1.7
Step 5 — Write the SRF Ŷ = 1.7 + 1.05X
Step 6 — Calculate Fitted Values and Residuals i=1: Ŷ = 1.7 + 1.05(2) = 3.8, ê = 4 − 3.8 = 0.2, ê² = 0.04 i=2: Ŷ = 1.7 + 1.05(4) = 5.9, ê = 6 − 5.9 = 0.1, ê² = 0.01 i=3: Ŷ = 1.7 + 1.05(6) = 8.0, ê = 8 − 8.0 = 0.0, ê² = 0.00 i=4: Ŷ = 1.7 + 1.05(8) = 10.1, ê = 9 − 10.1 = −1.1, ê² = 1.21 i=5: Ŷ = 1.7 + 1.05(10)= 12.2, ê = 13 − 12.2 = 0.8, ê² = 0.64 SSR = Σê² = 0.04 + 0.01 + 0 + 1.21 + 0.64 = 1.90
Step 7 — Calculate TSS TSS = Σyᵢ² = Σ(Yᵢ − Ȳ)² yᵢ² values: 16, 4, 0, 1, 25 TSS = 16 + 4 + 0 + 1 + 25 = 46
Step 8 — Calculate R² R² = 1 − SSR/TSS = 1 − 1.90/46 = 1 − 0.0413 = 0.9587
Final Answer: Ŷ = 1.7 + 1.05X R² = 0.9587 (or 95.87%)
Example 2: t-Test and Confidence Interval
Using the regression above: n = 5, β̂₁ = 1.05, SSR = 1.90, Σxᵢ² = 40. (a) Test H₀: β₁ = 0 vs H₁: β₁ ≠ 0 at α = 5%. (b) Construct a 95% confidence interval for β₁.
Step 1 — Calculate σ̂² df = n − 2 = 5 − 2 = 3 σ̂² = SSR / df = 1.90 / 3 = 0.6333
Step 2 — Calculate Var(β̂₁) Var(β̂₁) = σ̂² / Σxᵢ² = 0.6333 / 40 = 0.01583
Step 3 — Calculate SE(β̂₁) SE(β̂₁) = √0.01583 = 0.1258
Step 4 — Calculate t-statistic t = β̂₁ / SE(β̂₁) = 1.05 / 0.1258 = 8.347
Step 5 — Find Critical Value Two-tailed test at α = 5%, df = 3 t_crit = t(0.025, 3) = 3.182 (from t-table)
Step 6 — Decision |t-stat| = 8.347 > t_crit = 3.182 → REJECT H₀
Part (b) — Confidence Interval CI = β̂₁ ± t_crit × SE(β̂₁) CI = 1.05 ± 3.182 × 0.1258 CI = 1.05 ± 0.400 Lower: 1.05 − 0.400 = 0.650 Upper: 1.05 + 0.400 = 1.450
(a) t = 8.347 > 3.182 → Reject H₀ at 5%. β₁ is statistically significant. (b) 95% CI for β₁: [0.650 ; 1.450] Note: Zero is NOT in the interval — consistent with rejecting H₀.
Example 3: F-Test for Overall Significance
Using the same regression: ESS = TSS − SSR = 46 − 1.90 = 44.10, SSR = 1.90, n = 5, k = 1. Test the overall significance of the model at α = 5%.
Step 1 — State Hypotheses H₀: β₁ = 0 (model has no explanatory power) H₁: β₁ ≠ 0 (model is significant)
Step 2 — Calculate F-statistic F = [ESS/k] / [SSR/(n−k−1)] F = [44.10/1] / [1.90/(5−1−1)] F = 44.10 / (1.90/3) F = 44.10 / 0.6333 F = 69.64
Step 3 — Find Critical Value Numerator df = k = 1 Denominator df = n − k − 1 = 3 F_crit(1, 3) at 5% = 10.13 (from F-table)
Step 4 — Decision F-stat = 69.64 > F_crit = 10.13 → REJECT H₀
Verify: F = t² From Example 2, t = 8.347 t² = 8.347² = 69.67 ≈ 69.64 ✓ (small rounding difference)
F = 69.64 > 10.13 → Reject H₀. The model is statistically significant at 5%. X significantly explains variation in Y.
Example 4: Expected Value and Variance Calculation
A random variable X has the following probability distribution: P(X=1) = 0.2, P(X=2) = 0.3, P(X=3) = 0.4, P(X=4) = 0.1 Find E(X), E(X²), Var(X), and SD(X).
Step 1 — Check probabilities sum to 1 0.2 + 0.3 + 0.4 + 0.1 = 1.0 ✓
Step 2 — Calculate E(X) E(X) = Σ xᵢ · P(xᵢ) = 1(0.2) + 2(0.3) + 3(0.4) + 4(0.1) = 0.2 + 0.6 + 1.2 + 0.4 = 2.4
Step 3 — Calculate E(X²) E(X²) = Σ xᵢ² · P(xᵢ) = 1²(0.2) + 2²(0.3) + 3²(0.4) + 4²(0.1) = 1(0.2) + 4(0.3) + 9(0.4) + 16(0.1) = 0.2 + 1.2 + 3.6 + 1.6 = 6.6
Step 4 — Calculate Var(X) Var(X) = E(X²) − [E(X)]² = 6.6 − (2.4)² = 6.6 − 5.76 = 0.84
Step 5 — Standard Deviation SD(X) = √0.84 = 0.9165
E(X) = 2.4, Var(X) = 0.84, SD(X) = 0.9165
Example 5: Elasticity Calculation
A regression gives: Ŷ = 50 + 3.5X, where Y = quantity demanded (units) and X = price (Rands). At the mean values X̄ = 20 and Ȳ = 120, calculate the price elasticity of demand.
Step 1 — Identify the slope β̂₁ = 3.5 This is a Lin-Lin model, so we use the point elasticity formula.
Step 2 — Apply elasticity formula ε = β̂₁ × (X̄ / Ȳ) = 3.5 × (20 / 120) = 3.5 × 0.1667 = 0.583
Step 3 — Interpret ε = 0.583 (positive... for demand, check sign convention) |ε| = 0.583 < 1 → Inelastic demand
Price elasticity of demand ≈ 0.583. A 1% increase in price leads to a 0.583% increase in quantity (inelastic).
Example 6: Log-Log Model Interpretation
A regression gives: ln(Y) = 2.3 + 0.75·ln(X), where Y = expenditure and X = income. Interpret β̂₁ and state what type of elasticity this gives.
Identify Model Type Both Y and X are in log form → Log-Log model. β̂₁ directly gives the constant elasticity.
Interpretation β̂₁ = 0.75 = income elasticity of expenditure. A 1% increase in income is associated with a 0.75% increase in expenditure. Since 0.75 < 1, expenditure is income-inelastic (it rises, but less than proportionally).
Income elasticity = 0.75. A 1% rise in income leads to a 0.75% rise in expenditure. Expenditure is a necessity (income-inelastic) since ε < 1.

Elasticity Interpretation Templates

Inelastic Demand (|ε| < 1) "The price elasticity of demand is [ε]. Since |ε| < 1, demand is price inelastic. A 1% increase in price leads to a less than 1% decrease in quantity demanded ([ε]%). Consumers are relatively unresponsive to price changes."
